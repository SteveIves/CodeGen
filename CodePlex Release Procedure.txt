CodePlex Release Procedure
==========================

This document defines the procedure for distributing a new "release" of
CodeGen on CodePlex.

1.  Bump the version number in CodeGenEngine\codegen.def

2.  Bump version number in the CodeGenSetup project properties

3.  Build all projects in Visual Studio and test the .NET version

4.  Make sure that any new files have been added to the Workbench projects.

5.  Build with Traditional Synergy 32-bit Windows and execute tests

6.  Build with Traditional Synergy 32-bit Windows and execute tests

7. If time permits, build and test the Unix version.

8. If time permits, build and test the OpenVMS version.

9.  Verify that enhancements and new features are documented in the help file

10. Add change information to the change log in the help file

11. Check out the compiled help file, build a new help file, and re add to the
    Visual Studio solution (it gets removed when it is rebuilt)

12. Build new web help files

13. Upload web help files to the web site

14. Rebuild the Visual Studio solution in Release mode

15. Build the setup project

16. Test the new installer

17. Check everything in and note the new "change set" number

18. Zip the MSI file and setup.exe and rename as appropriate

19. Upload the zip file to CodePlex and publish the release, associating it
    with the latest change set number

20. Create next release as a "planned" release

21. Close any CodePlex "Issues" that are related to the new release

22. Update the associated release number in any issues that were slated for
    the release but didn't make it in
