FIELD_SQLNAME

 

Inserts a SQL-compatible name for the current field. Array fields are expanded to multiple individual fields, with the name being suffixed with the array element number, for example 'FIELDNAME1', 'FIELDNAME2' etc.

The token behaves identically to the <FIELD_NETNAME> token.

Usage

<FIELD_SQLNAME>

Alternate forms

•<field_sqlname>

•<Field_Sqlname>

•<Field_sqlname>

•<FieldSqlname>

•<fieldSqlname>

In addition, for compatibility with previous earlier versions, the following alternate forms are also supported but ideally should not be used:

•<FieldSqlName>

•<fieldSqlName>
